Arc Testnet proof

Arqis registry contract.

Arqis has an app-specific Arc Testnet contract for lightweight invoice and payment-reference proof. The current MVP remains non-custodial and testnet-only.

What this is

Proof registry, not custody.

The registry records invoice metadata hashes and payment transaction references for Arqis. It does not hold funds, execute swaps, bridge assets, or claim production payment verification.

Deployed on Arc Testnet
Why it exists

App-specific onchain footprint.

Arc reviewers can see that Arqis has its own contract and real interactions on Arc Testnet instead of pointing to infrastructure contracts like USDC or CCTP.

For the app creator

Credibility and audit trail.

This gives Arqis a project-owned contract address and transaction history that can be shared with Arc reviewers, GitHub readers, and future partners.

For web users

No extra burden today.

Users do not need private keys outside their wallet and do not need to interact with this registry in the current web MVP. Future versions can let sellers optionally record invoice proof from their connected wallet.

Scope note: The current web app demonstrates invoice creation, payer checkout, and Seller Console review. The registry is deployed and has a test invoice proof transaction; full web integration with the registry is planned next.
